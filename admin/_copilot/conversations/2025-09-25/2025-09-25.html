üìÑ Why Articles Deserve Paging First

Articles tend to:

  • Accumulate rapidly: Editorial teams publish frequently, so the volume grows fast.

  • Require browsing: Users often scroll through lists to find relevant reads, unlike blocks or categories which are more structural.

  • Support filters/search: Paging pairs well with keyword search, tags, or category filters‚Äîespecially in your modal assignment flow.

  • Impact performance: Loading hundreds of articles at once can slow things down, especially in modal-based UIs.

üß± What About Other Modules?

  • Pages: Usually fewer in number, more static‚Äîpaging might be overkill unless you're managing hundreds.

  • Blocks: Often reused components; better suited to grouped views or collapsible lists.

  • Categories: Typically a small, curated set‚Äîdropdowns or tree views work better than paging.

  • Add offset + limit to your SQL queries.

üß† Concept First: What We‚Äôre Adding

We‚Äôll introduce:

  • limit and offset in the SQL query

  • A way to pass the current page via $_GET['page']

  • Basic pagination controls (Prev / Next)

üîß Step-by-Step Implementation

1. Set Pagination Variables

Add this before your SQL query:

php

$limit = 10; // articles per page
$page = isset($_GET['page']) ? max(1, intval($_GET['page'])) : 1;
$offset = ($page - 1) * $limit;

2. Update SQL Query

Replace your current query with:

php

$sql = "SELECT * FROM articles ORDER BY sort ASC LIMIT $limit OFFSET $offset";

3. Get Total Count for Pagination

After the query block, add:

php

$countResult = $conn->query("SELECT COUNT(*) AS total FROM articles");
$totalArticles = $countResult->fetch_assoc()['total'];
$totalPages = ceil($totalArticles / $limit);

4. Render Pagination Controls

After the </table>, add:

php

<div style="margin-top:20px;">
  <?php if ($page > 1): ?>
    <a href="?page=<?php echo $page - 1; ?>">‚¨Ö Prev</a>
  <?php endif; ?>

  Page <?php echo $page; ?> of <?php echo $totalPages; ?>

  <?php if ($page < $totalPages): ?>
    <a href="?page=<?php echo $page + 1; ?>">Next ‚û°</a>
  <?php endif; ?>
</div>

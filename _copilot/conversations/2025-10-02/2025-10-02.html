function getCount($table) {
  global $pdo;
  $stmt = $pdo->query("SELECT COUNT(*) FROM $table WHERE status = 'on'");
  return $stmt->fetchColumn();
}

function getRecent($table, $titleCol, $dateCol, $limit = 5) {
  global $pdo;
  $stmt = $pdo->prepare("SELECT $titleCol, $dateCol FROM $table ORDER BY $dateCol DESC LIMIT ?");
  $stmt->execute([$limit]);
  return $stmt->fetchAll(PDO::FETCH_ASSOC);
}

Copilot said

You're running into two key issues here, Nasiruddin bhai:

⚠️ 1. Function Doesn't Return Anything

Your getRecent() function prints rows inside the loop but doesn’t return them. So when you later do:

php



$recentArticles = getRecent(...);

$recentArticles is null, which breaks the foreach loop.

✅ Fix: Return the Rows

Update your function like this:

php



function getRecent($table, $titleCol, $dateCol, $limit = 5) {
    global $conn;

    $sql = "SELECT $titleCol, $dateCol FROM $table ORDER BY $dateCol DESC LIMIT $limit";
    $result = $conn->query($sql);

    $rows = [];
    while ($row = $result->fetch_assoc()) {
        $rows[] = $row;
    }
    return $rows;
}

Now $recentArticles will contain an array of rows, and your foreach will work perfectly.

⚠️ 2. SQL Injection Risk

You're directly injecting $table, $titleCol, $dateCol, and $limit into the query string. This is dangerous if any of those values come from user input.

✅ Optional: Whitelist Columns and Tables

If you want to be safe and future-proof:

php

Copy

$allowedTables = ['articles'];
$allowedCols = ['title', 'entry_date_time'];

if (!in_array($table, $allowedTables) || !in_array($titleCol, $allowedCols) || !in_array($dateCol, $allowedCols)) {
    return [];
}
